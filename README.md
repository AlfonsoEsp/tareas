<h1>Objetivo del repositorio<h1>
<h2>El objetivo principal de este repositorio es con el fin de servir como bitácora digital como un portafolio que documente los trabajos y mi progreso académico en el ámbito de la programación web y diseño web. En este mismo espacio tiene como fin almacenar, organizar, y presentar los trabajos solicitados durante el curso de este trimestre, demostrando la aplicación y el aprendizaje de la práctica de los lenguajes que se vieron en el curso que son:HTML, CSS y JavaScript. Más con el simple hecho de almacenar código, en este repositorio busca reflejar la integración entre esta lógica computacional y la expresión creativa, donde se utiliza librerías gráficas como p5.js para la transformación de algoritmos abstractos en resultados visuales.
Asimismo, este repositorio tiene como finalidad académica demostrar el dominio de estas herramientas proporcionadas por el tutor durante el curso. En el entorno profesional actual, la capacidad de gestionar cambios, mantener un historial de trabajo y colaborar en la nube mediante GitHub es tan importante como el saber escribir código mismo. Por lo tanto, este proyecto no solo busca simplemente buscar cumplir los requisitos de evaluación de la UEA, sino también desarrollar una disciplina de trabajo donde cada actividad representa un paso adelante para la comprensión de la estructura web.

<h1>Organización del repositorio<h1>
<h2>La organización de este repositorio ha sido diseñada siguiendo las indicaciones del tutor y aplicando lo presentado en clases para desarrollar buenas prácticas en el desarrollo web, la estructura está basada en carpetas independientes para cada proyecto o asignación, lo que permite una navegación intuitiva para la mayoría de usuarios o cualquiera que visite el perfil. Donde cada carpeta se divide en aspectos de estructura del código y estilos.
Ejemplos como archivos HTML (como index.html o p5.html) que actúan como el esqueleto, proyectos o actividades donde se importan las librerías necesarias mediante CDNs. En segundo lugar, se ubica los archivos CSS(style.css), encargado exclusivamente de la presentación visual, el diseño y la estética del contenedor del canvas. Y finalmente el archivo JavaScript (sketch.js)contiene lógica de programación y las funciones de dibujo.. Esta separación facilita la depuración de errores y el mantenimiento del código a largo plazo. Además la nomenclatura de los archivos se han mantenido en minúsculas y sin espacios para asegurar la compatibilidad con los servidores y evitar conflictos en el despliegue de la web, con la finalidad de categorizar estos tipos de archivos y tener claro el funcionamiento de cada uno de estos mismos y así evitar posibles archivos no compatibles.<h2>
